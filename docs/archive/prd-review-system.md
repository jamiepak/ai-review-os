# AI 设计评审系统 — 产品需求文档（PRD）

> 版本: v1.0 | 日期: 2026-04-14
> 产品: AI Review OS · 评审预约与自动处理
> 项目: VooV Meeting 设计评审流程重构

---

## 一、背景与目标

### 1.1 现状问题

| 问题 | 现状 | 影响 |
|------|------|------|
| **人工驱动** | 评审登记靠人催、靠填表 | 效率低，易遗漏 |
| **信息碎片化** | TAPD、企微文档、表格各自独立 | 评审准备不充分 |
| **无自动通知** | 靠人在群里手动 @ 通知 | 常忘记、不及时 |
| **无内容审查** | 需求文档质量参差不齐 | 评审会上才发现问题 |

### 1.2 目标

构建一个 **AI 驱动的评审预约与内容处理系统**，实现：

1. **一键预约** — 粘贴 TAPD 链接即可完成评审登记
2. **自动提取** — AI 自动从 TAPD + 腾讯文档中提取评审内容
3. **自动通知** — 群内自动推送评审信息并 @ 相关人
4. **评审提醒** — 评审前自动提醒相关同学准备材料

### 1.3 核心指标

| 指标 | 目标 |
|------|------|
| 评审登记耗时 | 从 10 分钟 → **30 秒** |
| 评审内容完整度 | 从人工检查 → **AI 自动补全** |
| 通知覆盖率 | 从看群消息 → **100% 定向通知** |

│           ↓                                                  │
│  ┌─ 群内通知 ──────────────────────────────────────────┐     │

│                                                              │
└──────────────────────────────────────────────────────────────┘

### 4.2 需求内容提取决策树

```
TAPD 需求详情 (description)
    │
    ├── 有实质内容（>200字 且 非空模板）
    │       → 使用 TAPD description 作为评审内容
    │       → AI 生成摘要（500字以内）
    │
    └── 无内容 / 空模板 / 仅链接
            │
            ├── 包含腾讯文档链接（docs.qq.com/*）
            │       → 调用腾讯文档 API 读取内容
            │       → AI 生成摘要
            │
            ├── 包含企微文档链接（doc.weixin.qq.com/*）
            │       → 调用企微文档 MCP 读取内容
            │       → AI 生成摘要
            │
            └── 无任何文档链接
                    → 标记状态"⚠️ 内容待补充"
                    → 通知发起人补充材料
```

### F3. 需求内容智能提取

#### 5.3.1 内容判定逻辑

```python
def extract_review_content(tapd_description):
    """
    从 TAPD 需求描述中提取评审内容
    返回: (content, source)
    """
    # Step 1: 清洗 HTML，提取纯文本
    clean_text = strip_html(tapd_description)
    
    # Step 2: 判断是否有实质内容
    if has_substantial_content(clean_text):
        # 有内容 → 直接使用
        return (clean_text, "tapd")
    
    # Step 3: 搜索腾讯文档链接
    doc_links = extract_doc_links(tapd_description)
    
    if doc_links:
        # 找到文档链接 → 读取文档内容
        for link in doc_links:
            content = fetch_doc_content(link)
            if content:
                return (content, f"doc:{link}")
        
    # Step 4: 无内容可提取
    return (None, "empty")
```

#### 5.3.2 内容判定标准

| 判定 | 条件 |
|------|------|
| **有实质内容** | 纯文本长度 > 200 字，且不匹配空模板特征词 |
| **空模板** | 包含 "请在此处填写"、"TODO"、"待补充" 等特征词，或仅有标题无正文 |
| **仅链接** | 纯文本去掉 URL 后长度 < 50 字 |

#### 5.3.3 支持的文档链接格式

| 平台 | 链接格式 | 提取方式 |
|------|---------|---------|
| 腾讯文档 | `docs.qq.com/doc/xxx` | 腾讯文档 MCP `get_content` |
| 腾讯文档智能表 | `docs.qq.com/sheet/xxx` | 腾讯文档 MCP `get_content` |
| 企微文档 | `doc.weixin.qq.com/xxx` | 企微文档 MCP 读取 |

#### 5.3.4 内容处理

提取到内容后：
1. **去除格式噪音** — 清除 HTML 标签、空行、无意义格式
2. **AI 生成摘要** — 将长文档压缩为 500 字以内的结构化摘要
3. **标记来源** — 注明内容来自 TAPD / 腾讯文档 / 企微文档

### F6. 评审前定时提醒

#### 5.6.1 实现方案

| 方案 | 实现 | 说明 |
|------|------|------|
| **云函数定时触发** | SCF 定时触发器，每天 9:00 和 17:00 检查明天/今天的评审 | 推荐 |
| 本地 cron | WorkBuddy automation | 依赖本机运行 |

#### 5.6.2 定时逻辑

```
每天 9:00 执行:
  1. 查询智能表格中"日期 = 明天"的评审记录（需 access_token）
  2. 如果有 → 推送"评审提醒"到群

每天（评审日）开始前 1 小时:
  1. 查询智能表格中"日期 = 今天"的评审记录
  2. 推送"最终提醒"

备选方案（不需 access_token）:
  - 提交预约时将提醒时间写入云函数侧的缓存/存储
  - 用 SCF 定时器按时触发通知
```

**处理流程:**

```
1. 遍历 stories:
   a. 调 TAPD API 获取需求详情

```

## 八、非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 单次提交响应 < 10 秒（含 TAPD 查询 + 文档读取 + 写表格 + 通知） |
| **可用性** | 云函数自动扩缩容，无需运维 |
| **安全** | Webhook Key 不暴露在前端；云函数只允许企微域名的 Webhook 调用 |
| **兼容** | 移动端优先，支持 iOS/Android 企微内置浏览器 |
| **成本** | 全程在腾讯云免费额度内 |
