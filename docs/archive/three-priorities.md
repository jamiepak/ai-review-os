## 事项①：自动化评审流程

### 要做的事

```
提报需求 → 写入智能表格 → 群内通知 → @指定同学准备评审
```

### 拆解

| 子步骤 | 实现方式 | 可行性 |
|--------|---------|--------|
| 用户提报需求 | 移动端页面（已有） | ✅ 已完成 |
| 写入智能表格 | Webhook add_records | ✅ 已验证 |
| **评审群通知** | 企微群机器人 Webhook | ✅ 可行 |
| **@指定同学** | 群机器人消息 + mentioned_list | ✅ 可行 |

### 关键技术：企微群机器人 Webhook

企微群里添加一个机器人，拿到 Webhook URL 后就能推消息，**不需要任何审批或 corpid/corpsecret**：

```bash
# 发送群消息（Markdown 格式）
curl 'https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_BOT_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "msgtype": "markdown",
    "markdown": {
      "content": "## 📋 新评审需求已登记\n> **需求**: 通过API创建用户并验证手机号\n> **产品**: 张三\n> **设计师**: 李四\n> **FT**: 企微专项\n\n<@userid1> <@userid2> 请准备评审材料"
    }
  }'
```

**@指定人**的方式：
- Markdown 消息中用 `<@userid>` 可以 @ 具体人
- 也可以用 `mentioned_list` / `mentioned_mobile_list` 字段

### 完整流程

```
用户在页面提交 TAPD 链接
        ↓
云函数处理:
  1. 调 TAPD API 提取需求信息
  2. 写入智能表格（Webhook）
  3. 推送群消息（群机器人 Webhook）
     - 评审需求摘要
     - @产品 @设计师 准备材料
     - 评审时间/日期
        ↓
群内收到通知，相关同学开始准备
```

### 需要你提供

| 信息 | 用途 | 获取方式 |
|------|------|---------|
| **评审群机器人 Webhook URL** | 推送群通知 | 群设置 → 添加机器人 → 复制 Webhook 地址 |
| **需要 @ 的人的 userid** | 精准提醒 | 企微管理后台可查，或用手机号代替 |

### 可行性：✅ 完全可行，零额外审批

## 事项②：会议录制按需求切分

### 要做的事

```
会议录制 → 获取转写文本 → 按需求关键词切分 → 每个需求绑定对应片段
```

### 技术路径分析

#### 路径 A：腾讯会议 REST API（理想方案）

| 步骤 | API | 状态 |
|------|-----|------|
| 获取录制列表 | `GET /v1/records` | ⚠️ 需 AppId + SecretKey |
| 获取转写文本 | `GET /v1/records/transcripts/{id}` | ⚠️ 同上 |
| AI 摘要 | `GET /v1/meetings/{id}/ai-summary` | ⚠️ 需企业版 |

**卡点**：需要腾讯会议开放平台的开发者凭证。

#### 路径 B：手动上传 + AI 处理（过渡方案）

如果暂时拿不到 API 权限，可以先用过渡方案：

```
会议结束 → 下载录制文件 → 上传到系统
                               ↓
             AI 自动处理:
             1. 语音转文字（用现有 ASR 能力）
             2. 按需求关键词分段
             3. 每个需求 → 时间区间 + 文字片段 + AI 摘要
```

| 步骤 | 实现 | 可行性 |
|------|------|--------|
| 下载录制 | 腾讯会议客户端手动下载 | ✅ 无依赖 |
| 语音转文字 | 腾讯云 ASR API / 本地 Whisper | ✅ 可行 |
| 按需求切分 | AI 语义分段（对照评审列表） | ✅ 可行 |
| 生成摘要 | AI 对每段生成概述和结论 | ✅ 可行 |

#### 路径 C：直接用腾讯会议已有的 AI 纪要（最省力）

腾讯会议本身已有 AI 纪要功能（会中/会后自动生成）。如果你们用的版本支持：

```
会议结束 → 腾讯会议 AI 纪要已生成
                    ↓
           手动复制/导出 AI 纪要文本
                    ↓
           粘贴到系统 → AI 按需求切分 → 绑定到表格
```

### 切分算法

不管哪条路径，最终都需要把一整段文本按需求切分。核心逻辑：

```
输入:
  - 转写/纪要文本（带时间戳）
  - 本场评审的需求列表（从智能表格获取）

处理:
  1. 对文本做段落分割
  2. 每个段落与需求列表做关键词/语义匹配
  3. 相邻属于同一需求的段落合并
  4. 输出: 需求ID → [开始时间, 结束时间, 文本片段]

输出:
  - 每个需求对应的讨论片段
  - 每个需求的 AI 评审概述
  - 每个需求的 AI 评审结论
```

### 推荐策略

| 阶段 | 方案 | 依赖 |
|------|------|------|
| **现在** | 路径 C（复制 AI 纪要）或路径 B（手动上传录制） | 无 |
| **拿到 API 后** | 路径 A（全自动） | AppId + SecretKey |

### 可行性：✅ 可行（路径 B/C 无额外依赖，路径 A 需凭证）

## 事项③：评审内容提取与 AI Review

### 要做的事

```
读取 TAPD 需求内容 + 读取需求中贴的企微文档 → AI 做预评审 Review
```

这是三件事里**最有技术含量也最有价值**的一个。拆解：

### 3a. 读取 TAPD 需求内容

| 数据 | 获取方式 | 状态 |
|------|---------|------|
| 需求名称、描述、创建人 | TAPD MCP API `stories_get` | ✅ 已验证 |
| 需求详细描述（富文本） | `stories_get` 的 `description` 字段 | ✅ 可获取 |
| 需求附件 | TAPD MCP API `get_attachment_info` | ✅ 可调用 |
| 需求评论 | TAPD MCP API `comments_get` | ✅ 可调用 |

**注意**：TAPD 需求的 description 是 HTML 富文本，需要做清洗提取纯文本。

### 3b. 读取企微文档内容

需求描述中可能贴了企微文档链接（如 `doc.weixin.qq.com/xxx`），需要自动提取并读取内容。

#### 技术方案对比

| 方案 | 实现 | 可行性 | 优劣 |
|------|------|--------|------|
| **A. 企微文档 API** | 调 `wedoc/get_doc` 接口获取文档内容 | ⚠️ 需 access_token | 最干净，结构化数据 |
| **B. 腾讯文档 MCP** | 用 `tencent-docs` skill 的 `get_content` | ⚠️ 需腾讯文档 Token，且仅支持 docs.qq.com | 仅限腾讯文档链接 |
| **C. 网页抓取** | 用 Playwright 打开链接提取内容 | ⚠️ 需登录态 | 通用但不稳定 |
| **D. 手动粘贴** | 用户在提报页面直接粘贴文档内容 | ✅ 无依赖 | 最简单，过渡方案 |

#### 推荐策略

**先 D 后 A**：
- 现在：页面增加「评审内容」文本框，用户可以直接粘贴 PRD/设计文档的核心内容
- 之后：拿到企微 access_token 后，自动从 doc.weixin.qq.com 链接抓取内容

#### 企微文档 API 调用方式（拿到凭证后）

```
1. 从 TAPD 需求描述中提取 doc.weixin.qq.com 链接
2. 解析出 docid
3. 调 API:
   POST https://qyapi.weixin.qq.com/cgi-bin/wedoc/get_doc
   {
     "docid": "xxx",
     "access_token": "xxx"
   }
4. 返回文档纯文本内容
```

### 3c. AI Review 评审

拿到内容后，AI 做自动预评审：

```
输入:
  - TAPD 需求信息（名称、描述、创建人、优先级）
  - 企微文档/PRD 内容
  - 评审 Checklist（知识库）
  - 历史失败模式（知识库）
  - FT 画像（知识库）

处理:
  AI 对照 Checklist 逐项检查，匹配失败模式

输出（预评审报告）:
  ┌─────────────────────────────────┐
  │ 📋 预评审报告                    │
  │                                  │
  │ 需求: XXX                        │
  │ FT: 企微专项                     │
  │                                  │
  │ 🔴 高风险项:                     │
  │ - 缺少异常态设计（命中失败模式#1）│
  │ - 无数据支撑（命中失败模式#2）   │
  │                                  │
  │ 🟡 建议改进:                     │
  │ - 补充竞品对比                   │
  │ - 明确成功指标量化方式           │
  │                                  │
  │ ✅ 已满足:                       │
  │ - 核心流程完整                   │
  │ - 有设计稿链接                   │
  │                                  │
  │ 预测通过率: 65%                  │
  │ 建议: 补充异常态设计后再提交评审 │
  └─────────────────────────────────┘
```

### 可行性汇总

| 子功能 | 可行性 | 当前可做 | 需凭证后 |
